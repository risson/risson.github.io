Since 2 weeks ago we've worked on a project (triggered email). The first thing we need to do is to clean up our database. I won't describe much detail about how messy the database is, but I believe it's one of the most horrible enterprise database in the world... There're lots of duplicate information in it, including email address and phone number, and clearly no one ever put any effort in cleaning those. We subset some basic data for the last 3 months, and want to see how many duplicate emails/phone numbers that we have. I wasn't able to find a way to work on it in R 2 weeks ago, but now I've found out the solution using the **data.table** package and the **dplyr** package. Below I'll illustrate how to do it by making some dummy data: ``` library(data.table) DT <- data.table(email = c("aaa@aaa.com","bbb@bbb.com","1123@qq.com","aaa@aaa.com","bbb@bbb.com"), status = c("student","student","lead","lead","opportunity")) DT ``` This is how it looks: ``` email status 1: aaa@aaa.com student 2: bbb@bbb.com student 3: 1123@qq.com lead 4: aaa@aaa.com lead 5: bbb@bbb.com opportunity ``` Then we only need to run 1 line of code: ``` DT[,count:=.N, list(email)] DT ``` ``` email status count 1: aaa@aaa.com student 2 2: bbb@bbb.com student 2 3: 1123@qq.com lead 1 4: aaa@aaa.com lead 2 5: bbb@bbb.com opportunity 2 ``` It automatically assigned the number of existences in a new column. If we want to see a summary by each email and sort by high frequency to low frequency: ``` DT2<-unique(DT, by="email") DT2 <- DT2[,list(email,count)] library(dplyr) DT2 <- arrange(DT2, desc(count)) DT2 ``` Here we use the **dplyr** package to help arrange the order by count. Below is the result of the above code: ``` email count 1: aaa@aaa.com 2 2: bbb@bbb.com 2 3: 1123@qq.com 1 ``` Please feel free to correct me if there's a flaw in my code. Your comments are appreciated :)

Since 2 weeks ago we've worked on a project (triggered email). The first thing we need to do is to clean up our database. I won't describe much detail about how messy the database is, but I believe it's one of the most horrible enterprise database in the world... There're lots of duplicate information in it, including email address and phone number, and clearly no one ever put any effort in cleaning those.

We subset some basic data for the last 3 months, and want to see how many duplicate emails/phone numbers that we have. I wasn't able to find a way to work on it in R 2 weeks ago, but now I've found out the solution using the data.table package and the dplyr package.

Below I'll illustrate how to do it by making some dummy data:

library(data.table)
DT <- data.table(email = c("aaa@aaa.com","bbb@bbb.com","1123@qq.com","aaa@aaa.com","bbb@bbb.com"), status = c("student","student","lead","lead","opportunity"))
DT

This is how it looks:

         email      status
1: aaa@aaa.com     student
2: bbb@bbb.com     student
3: 1123@qq.com        lead
4: aaa@aaa.com        lead
5: bbb@bbb.com opportunity

Then we only need to run 1 line of code:

DT[,count:=.N, list(email)]
DT

         email      status count
1: aaa@aaa.com     student     2
2: bbb@bbb.com     student     2
3: 1123@qq.com        lead     1
4: aaa@aaa.com        lead     2
5: bbb@bbb.com opportunity     2

It automatically assigned the number of existences in a new column. If we want to see a summary by each email and sort by high frequency to low frequency:

DT2<-unique(DT, by="email")
DT2 <- DT2[,list(email,count)]
library(dplyr)
DT2 <- arrange(DT2, desc(count))
DT2

Here we use the dplyr package to help arrange the order by count. Below is the result of the above code:

         email count
1: aaa@aaa.com     2
2: bbb@bbb.com     2
3: 1123@qq.com     1

Please feel free to correct me if there's a flaw in my code. Your comments are appreciated :)
